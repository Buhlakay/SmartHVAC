Home Introduction Project Description Goals Processes

Introduction

 In recent years many people are buying HVAC units to help with heating and cooling their home. With home automation becoming more and more popular and systems like the Nest gaining popularity, we thought that smart HVAC stystems were missing something. Much of the electrical bill for a household comes from heating and cooling, a lot of times throughout the day when the owner isn't even home. Often times it can be difficult to remember to turn of your HVAC system or keep it on a consistent schedule when people's schedule's are constantly changing.

Project Description

 We wanted to help combat rising temperatures and electrical costs with our system. Our system is a smart HVAC system that you will be able to sync to your own schedule so that it will automatically heat and cool your home to the optimal temperature throughout the day in depending on how the outside temperature changes throughout the day. Our system will adapt to the temperature in order to heat and cool your home accordingly and save you money.

 However, if unforseen circumstances require you to defer from your normal schedule, the system will automatically change back to your own preferred temperature whenever it detects movement at home.

Goals

There were a few goals we knew we had to accomplish:

  • Develop smart HVAC system that will automatically adjust heating/air conditioning settings
  • Make decisions based on outside temperature and presence
  • Ensure that the system can be constantly running with little to no downtime
  • Allow users to customize temperature settings to fit their personal needs
  • Allow users to see that the system is working and what it has been doing over time

Processes

IBM Bluemix Cloud

 Used to run script that gets values from the raspberry pi. Creates and displays graph that shows the temperature within the house throughout the day. Graph shows when the heating or cooling has been turned on

Raspberry Pi

 The Raspberry Pi is responsible for retrieving values from the PIR motion sensor and retrieving current temperature from the Dark Sky API. It publishes to a sensorValues topic/event on the IBM Cloud broker so that the other components of the smart HVAC system can retrieve them and make decisions based on the current values. In practice, we would expect that the Raspberry Pi is placed somewhere so that the motion sensor would be able to "see" the entire room, so that it can accurately notify the system whether someone is present. One of the limitations of our motion sensor is that it only detects current motion, not presence. This means that if a person is sitting still in a room, the sensor would report that there is no motion. We overcame this limitation through the use of a time threshold. The code in the Raspberry Pi will count for 60 seconds (this time threshold can be changed) after motion has been detected and if no motion is detected within that 60 seconds, we assume that the room is empty.

Software Development

We made use of a few APIs to accomplish our goals:

  • Dark Sky
  • Google Calendar
 We used Dark Sky for all of our weather data. We collect data from Dark Sky based on a set zip code and then sample this throughout the day in order to change the temperature in the household.
 We used Google Calendar to collect all of the schedule information. The user can upload their daily schedule to Google Calendar and our system will know when the user will or won't be home so that it can heat or cool the home in order to best match with the outside temperature.

Hardware Development

 We used a Raspberry Pi for out main hardware controller. The Pi had a few functions for this project. First it is a publisher for the MQTT broker and handles the motion sensor logic. The Pi will run code to process the motion sensor values and then publish them to the MQTT server to be read. We also decided to have the Pi retrieve and process the weather values from the Dark Sky api.
